# mozambik_3Dmodels_library

# Мозамбик

Платформа бесплатных 3D-моделей

Учебный демонстрационный проект, изначально развёрнутый на хостинге InfinityFree и перенесённый в репозиторий для сохранения полной архитектуры, визуальной системы, документации и рабочего цикла администрирования.

## О проекте

Мозамбик — это веб-приложение для каталогизации, просмотра и скачивания бесплатных 3D-моделей с футуристичным дизайном в стиле киберпанк. Проект ориентирован на дизайнеров интерьера, 3D-художников, геймдев-разработчиков и студентов, которым требуется быстрый доступ к готовым ассетам без платных подписок или сложных процедур авторизации.

Платформа решает три ключевые задачи:
- Предоставляет удобный каталог с поиском, фильтрацией по категориям и форматам файлов.
- Обеспечивает интерактивный просмотр моделей прямо в браузере через Google Model Viewer.
- Даёт администратору полный контроль над контентом: добавление, редактирование, замена файлов и удаление моделей с автоматической очисткой дискового пространства.

Проект реализован по классической трёхзвенной архитектуре с разделением статической вёрстки и динамической логики. Статические HTML-файлы содержат подробные комментарии [DEV HINT], описывающие точки интеграции с серверной частью, что упрощает адаптацию кода под разные окружения или фреймворки.

## Архитектура и стек

Бэкенд:
- PHP 7.2+ с использованием PDO и подготовленных выражений для защиты от SQL-инъекций.
- Сессионная авторизация с хешированием паролей через password_hash / password_verify.
- Валидация и обработка множественных загрузок файлов с проверкой расширений и генерацией уникальных имён.
- Автоматическая генерация URL-дружественных идентификаторов (slug) на кириллице и латинице.

Фронтенд:
- HTML5 семантическая разметка, CSS3 с переменными, Grid, Flexbox, backdrop-filter, неоновые градиенты и анимации.
- Vanilla JavaScript: параллакс-отслеживание курсора, IntersectionObserver для появления секций, 3D-наклон карточек (tilt), управление модальными окнами и мобильным меню.
- Поддержка адаптивности через брейкпоинты 1200px, 992px, 768px, 480px с перестроением сеток и скрытием второстепенных элементов.

3D-рендеринг:
- @google/model-viewer (CDN) для отображения GLB/GLTF-моделей с управлением камерой, автоповоротом, тенями и нейтральным окружением.
- Фоллбэк на статичное изображение, если 3D-файл не загружен.

Дизайн:
- Шрифты: Orbitron (заголовки), Rajdhani (основной текст).
- Киберпанк-эстетика: неоновые градиенты (#7b2fff → #00ffff), glassmorphism, анимированные фоновые свечения, шумовые текстуры.
- Тёмная тема с акцентными элементами и микроанимациями при взаимодействии.

Сборка и деплой:
- Проект не требует сборщиков или пакетных менеджеров. Работает на любом LAMP-стеке или локальном PHP-сервере.
- Статическая версия полностью автономна и может размещаться на GitHub Pages или статических хостингах.

## Структура репозитория

admin/ — административная панель
- index.php — таблица всех моделей с превью, статусом 3D-файлов, датой создания и кнопками редактирования/удаления.
- model-create.php — форма добавления модели с валидацией полей, проверкой форматов файлов, генерацией slug и записью в БД.
- model-edit.php — форма редактирования с отображением текущих файлов, возможностью частичной замены медиа и безопасным обновлением записи.
- delete-model.php — удаление модели и связанных файлов с диска, редирект в список.
- login.php / logout.php — вход по логину/паролю, проверка хеша, управление сессией, защита от доступа без авторизации.

css/ — стили
- style.css — основной набор переменных, компоненты главной страницы, каталога, карточки товара, футера, адаптивные сетки и анимации в стиле киберпанк.
- admin.css — тёмная тема панели управления, формы, таблицы, кнопки, алерты, полная мобильная адаптация дашборда.
- about.css, contacts.css, docs.css — стили информационных страниц с единой дизайн-системой, различия в компоновке секций и типографике.

database/ — структура данных
- if0_41491682_mozambik.sql — полный дамп: создание таблиц admins и models, индексы, автоинкременты, кодировка utf8mb4, стартовые записи моделей.

includes/ — общие модули
- db.php — подключение к базе через PDO, настройка режима ошибок и формата выборки.
- auth.php — проверка сессии администратора, редирект на страницу входа при отсутствии авторизации.
- functions.php — вспомогательные функции: e() для экранирования, makeSlug() для генерации ЧПУ.

uploads/ — пользовательские файлы

*.html — статические версии страниц
- index.html, catalog.html, product.html, about.html, contacts.html, privacy.html, terms.html, consent.html, copyright.html — статические версии страниц с комментариями для интеграции.

*.php — динамические версии страниц
- catalog.php, product.php — страницы с подключением к базе данных и динамической подгрузкой контента.

## База данных

Таблица admins
- id: первичный ключ, автоинкремент.
- username: уникальный логин администратора.
- password_hash: хеш пароля, созданный функцией password_hash().
- created_at: время создания учётной записи.

Таблица models
- id: первичный ключ, автоинкремент.
- title: название модели, отображается в каталоге и карточке.
- slug: уникальный URL-идентикатор, генерируется из названия.
- description: полное описание, хранится как MEDIUMTEXT, поддерживает переносы строк.
- category: категория модели, используется для фильтрации и блока "Похожие модели".
- format: основной формат файла (FBX, OBJ, GLTF и т.д.).
- preview_image, product_image, model_file, viewer_file: относительные пути к загруженным файлам.
- is_featured: флаг популярности (0/1), зарезервировано для будущих подборок.
- created_at: время добавления модели.

Индексы: уникальные ключи на slug и username, что гарантирует отсутствие дубликатов и ускоряет поиск. Кодировка utf8mb4 обеспечивает корректную работу с кириллицей и специальными символами.

## Административная панель

Рабочий цикл администратора:
1. Авторизация через login.php. Система проверяет соответствие логина и хеша пароля, создаёт сессию.
2. Доступ к index.php защищён includes/auth.php. Без валидной сессии происходит редирект.
3. Добавление модели: форма принимает текстовые данные и до четырёх файлов. Скрипт проверяет обязательность полей, валидирует расширения, создаёт уникальные имена через uniqid(), перемещает файлы в uploads/ и выполняет INSERT в БД.
4. Редактирование: при частичной замене файлов старые удаляются с диска через unlink(), новые загружаются, запись обновляется через UPDATE.
5. Удаление: файл превью и модельный файл удаляются с сервера, запись убирается из таблицы.

Безопасность:
- Все SQL-запросы используют подготовленные выражения PDO.
- Пароли никогда не хранятся в открытом виде.
- Загрузка файлов ограничена белыми списками расширений.
- Генерация slug исключает символы, опасные для URL.
- Сессии очищаются при выходе, редирект предотвращает доступ к кэшированным страницам после логаута.

## Фронтенд и визуальная система

Дизайн построен на принципах киберпанк-эстетики, тёмной темы и микроанимаций:
- Фон: многослойные радиальные градиенты (#7b2fff, #00ffff) с эффектом размытия, шумовая текстура через SVG-фильтр, анимированное неоновое свечение.
- Карточки: полупрозрачные фоны с backdrop-filter, внутренние тени, скруглённые углы, эффекты при наведении (подъём, усиление свечения, масштаб изображений).
- Типографика: Orbitron для заголовков, Rajdhani для основного текста, вариативные размеры через clamp().
- Интерактивность: плавное появление секций при скролле, параллакс-эффекты фона, наклон карточек в 3D-пространстве, модальные окна с закрытием по Escape и клику вне области.

Адаптивность реализована через постепенное упрощение сеток: от трёх-четырёх колонок на десктопе до одной на мобильных устройствах. Критические элементы перестраиваются без потери читаемости и интерактивности.

## Установка и запуск

Требования:
- PHP 7.2 или выше, включены расширения pdo_mysql, mbstring, fileinfo.
- MySQL 5.7+ или MariaDB 10.3+.
- Веб-сервер Apache или Nginx с поддержкой обработки PHP.
- Права на запись в директорию uploads/.

Последовательность развёртывания:
1. Создать базу данных: CREATE DATABASE mozambique CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
2. Импортировать структуру: mysql -u пользователь -p mozambique < database/butterfly.sql
3. Настроить подключение в includes/db.php, указав хост, имя базы, логин и пароль.
4. Установить права на папки: chmod -R 755 uploads/, убедиться, что веб-сервер может писать в uploads/previews/, uploads/products/, uploads/files/, uploads/viewers/.
5. Создать администратора, сгенерировав хеш пароля через password_hash('ваш_пароль', PASSWORD_DEFAULT) и вставив запись в таблицу admins.
6. Запустить проект через встроенный сервер php -S localhost:8000 или настроить виртуальный хост.
7. Открыть /admin/login.php для входа в панель управления.

## Интеграция статики и бэкенда

Репозиторий содержит как статические HTML-файлы, так и динамические PHP-скрипты. Для перехода к рабочей версии:
- Заменить ссылки catalog.html на catalog.php, product.html на product.php в навигации и карточках.
- Убедиться, что форма поиска передаёт параметры search и category через GET.
- В карточках товаров ссылки должны вести на product.php?slug=значение, где slug извлекается из БД.
- Статические заглушки изображений заменить на динамические пути из базы данных.
- Комментарии [DEV HINT] содержат точные указания по экранированию, обрезке текста и обработке пустых состояний.
